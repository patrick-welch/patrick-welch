# Hi, I'm Patrick Welch

I’m an embedded systems and digital design developer focused on **measurement, control, and compute**.

I’m currently completing a **B.S. in Computer Science at Colorado State University** while building a portfolio in:

- FPGA / RTL design
- embedded systems
- RISC-V software and hardware
- verification and simulation
- scientific and hardware-adjacent computing

My goal is to pivot from line-of-business software into work closer to physical systems, instrumentation, digital design, and technical computing.

---

## Focus Areas

### FPGA & RTL Design
- Developing HDL projects in **Vivado** on the **Digilent Basys 3**
- Building and testing softcore-based systems with **PicoRV32** and **VexRiscv**
- Using **Verilator**, **VS Code**, and simulation-driven workflows for development and verification
- Exploring **Vitis HLS** and hardware/software co-design

### Embedded & RISC-V Systems
- Writing embedded **C** for both softcores and native RISC-V hardware
- Using Linux VMs for cross-compilation, build automation, and toolchain work
- Running Debian on the **VisionFive 2** as a native RISC-V target
- Developing toward complete SoC-style projects that combine firmware, HDL, and verification

### Scientific / Technical Computing
- Interested in systems where software meets hardware, measurement, and real-world constraints
- Drawn to numerical thinking, simulation, verification, and compute-heavy engineering workflows
- Especially interested in practical computing for instrumentation, control, and physical systems

### ASIC Toolchains (Exploratory)
- Exploring **OpenLane**, **Sky130**
- Using simple RTL blocks and small designs to better understand synthesis, flows, and pre-silicon thinking
- Not targeting physical design specialization directly, but using these tools to deepen RTL and system-level engineering skills

### Retro Hardware & Reimplementation
- Studying Atari 68K systems and related HDL-based reimplementation efforts
- Interested in legacy logic, bus behavior, MMU/GLUE-era design, and reconstruction of classic hardware behavior
- Using retro platforms as a serious learning vehicle for digital design and embedded systems

### Measurement & Bench Practice
- Strong interest in instrumentation, test equipment, and bench methodology
- I value careful measurement, documentation, repair, and repeatable workflows
- Good bench practice is part of how I think about engineering
